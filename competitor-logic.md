# 경쟁사 편성 계산 로직

## 원칙

- 경쟁사 편성 페이지의 모든 지표는 프론트에서 계산한다.
- Firebase RTDB는 월별 원본 데이터 저장소로만 사용한다.
- 프론트는 `competitorInfo/monthly/{YYYYMM}` 데이터를 읽고, 같은 월 데이터는 `localStorage`에 캐시한다.

## 필터

- `월`: 현재 선택된 `YYYYMM`.
- `MD분류`: 선택된 카테고리. `전체`이면 월 전체 데이터를 사용한다.
- 날짜 필터는 경쟁사 편성 페이지 계산에서 사용하지 않는다.

## 브랜드 집계

1. 회사별 원본 행을 읽는다.
   - `SSG`
   - `K`
   - `SK`
2. 선택된 `MD분류`에 해당하는 행만 회사별로 필터링한다.
3. 같은 브랜드의 가중분을 합산한다. 
4. 브랜드별 가중분 내림차순으로 정렬한다.


가중분은 아래 우선순위로 사용한다.

1. `brandWeightedMin`
2. `weightedMin`
3. `simpleMin`

## 편성비중

편성비중은 카테고리 안에서 회사끼리 나누는 비중이 아니다.

회사별로 아래 공식으로 계산한다.

```text
편성비중 = 해당 회사의 선택 MD분류 가중분 합 / 해당 회사의 월 전체 가중분 합 * 100
```

예시:

```text
SSG 보험 편성비중 = SSG의 보험 가중분 합 / SSG의 202606 전체 가중분 합
K 보험 편성비중   = K의 보험 가중분 합 / K의 202606 전체 가중분 합
SK 보험 편성비중  = SK의 보험 가중분 합 / SK의 202606 전체 가중분 합
```
## 운영여부

선택된 `MD분류`의 브랜드 집계를 기준으로 계산한다.

- `SSG`, `K`
  - 해당 브랜드가 `SK` 브랜드 목록에 있으면 `당사운영여부 = Y`
  - 없으면 `N`
- `SK`
  - 해당 브랜드가 `SSG` 또는 `K` 브랜드 목록에 있으면 `경쟁사운영여부 = Y`
  - 없으면 `N`

## 회사별 브랜드수

선택된 `MD분류`에서 회사별로 집계된 고유 브랜드 수다.

```text
브랜드수 = 선택 MD분류 내 회사별 고유 브랜드 개수
```

## 전월대비증감

전월대비증감은 화면을 지저분하게 만들지 않기 위해 회사 카드의 툴팁으로 표시한다.

비교 대상은 현재 선택 월의 직전 월이다.

```text
예: 202606 선택 시 202605와 비교
```

전월대비증감도 프론트에서 계산한다.

1. 현재 월 원본 데이터와 전월 원본 데이터를 RTDB 또는 localStorage 캐시에서 읽는다.
2. 현재 선택된 `MD분류` 기준으로 회사별 행을 필터링한다.
3. 회사별로 아래 값을 계산한다.
   - `PGM수`: 날짜, 시간, 브랜드, 상품명 조합 기준의 고유 편성 수
   - `가중분`: 선택 MD분류 내 가중분 합
   - `매출액`: 선택 MD분류 내 매출 합
4. 현재 월 값에서 전월 값을 뺀다.

```text
PGM수 증감 = 현재월 PGM수 - 전월 PGM수
가중분 증감 = 현재월 가중분 합 - 전월 가중분 합
매출액 증감 = 현재월 매출액 합 - 전월 매출액 합
```
